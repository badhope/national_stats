# 项目文件结构说明

## 目录概览

```
national_stats/
│
├── 📁 docs-presentation/          # 静态展示网页包（纯展示，无功能代码）
│   │                                 # 仅包含HTML、CSS、基础JS用于页面展示
│   │                                 # 不包含任何业务逻辑或功能实现
│   │
│   ├── 📁 app/                      # 功能程序包（可独立运行）
│   │                                 # 包含实际业务逻辑和功能实现
│   │                                 # 可在用户本地环境独立运行
│   │
├── 📁 core/                         # 核心功能模块库
│                                     # 被 app/ 调用的功能模块
│
├── 📁 .github/                      # GitHub 配置
│
└── 📄 README.md                     # 项目说明
```

## 文件类型说明

### 🔵 类型一：静态展示网页文件 (docs-presentation/)

**用途**：仅用于向用户展示和介绍项目功能、外观和特点  
**特点**：
- 纯静态 HTML/CSS/JavaScript
- 不包含任何实际业务逻辑
- 不执行任何数据处理
- 仅供展示和说明

**命名规范**：
- 目录前缀：`docs-presentation-`
- 文件前缀：`presentation-`
- 文件后缀：`.html`, `.css`, `.js`
- 注释标记：`<!-- 静态展示用 - 仅包含展示内容 -->`

**示例**：
```
docs-presentation/
├── index.html                    # 展示首页
├── presentation-style.css        # 展示样式
├── presentation-demo.js          # 演示脚本
└── README.txt                    # 说明文件
```

### 🟢 类型二：功能程序文件 (app/)

**用途**：实现实际功能逻辑，可在本地独立运行  
**特点**：
- 包含完整业务逻辑
- 可处理数据、执行分析
- 可在本地环境运行
- 独立于静态网页

**命名规范**：
- 目录前缀：`app-`
- 文件前缀：`app_` 或 `module_`
- 文件后缀：`.py`, `.js`, `.sh`
- 注释标记：`# 功能程序 - 实际业务逻辑`

**示例**：
```
app/
├── main.py                       # 程序入口
├── analyzer.py                   # 分析器
├── runner.sh                     # 运行脚本
└── requirements.txt              # 依赖
```

### 🟡 类型三：核心模块 (core/)

**用途**：被功能程序调用的底层模块库  
**特点**：
- 提供基础功能支持
- 被 app/ 引用
- 可被多个程序共享

**命名规范**：
- 目录名称：`core`
- 无特殊前缀
- 注释标记：`# 核心模块`

## 文件分离原则

### 1. 绝对隔离
```
静态展示 ────── ✗ ────── 功能程序
    │                            │
    ✗ 无文件引用                ✗ 无逻辑关联
    ✗ 无数据交换                ✗ 无路径依赖
    ✗ 无配置共享                ✗ 无函数调用
```

### 2. 独立运行
- **静态网页**：双击 HTML 文件即可查看（无需服务器）
- **功能程序**：`python app/main.py` 独立运行

### 3. 零依赖
- 静态网页不引用任何功能程序文件
- 功能程序不依赖静态网页文件

## 快速开始

### 查看静态展示网页
```bash
# 方式1：直接打开
双击 docs-presentation/index.html

# 方式2：本地服务器
cd docs-presentation
python -m http.server 8080
# 访问 http://localhost:8080
```

### 运行功能程序
```bash
# 安装依赖
pip install -r app/requirements.txt

# 运行程序
python app/main.py
```

## 注意事项

⚠️ **重要提示**：

1. **静态展示网页不包含任何功能代码**
   - 仅供展示和介绍使用
   - 按钮、表单等仅为视觉效果
   - 不执行任何实际业务逻辑

2. **功能程序独立于网页运行**
   - 需要安装依赖后才能运行
   - 不依赖静态网页文件
   - 可在命令行独立执行

3. **两者完全分离**
   - 无任何文件引用关系
   - 无任何数据交换
   - 无任何逻辑关联
